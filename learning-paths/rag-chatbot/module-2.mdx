## 2.1 Repository Setup and Dependencies

  <Card title="UI Components" icon="layout">

### Repository Structure

```
ai-sdk-rag-starter/
├── app/                    # Next.js App Router pages
├── components/             # React components
├── lib/                    # Utility functions and configurations
│   ├── db/                # Database configuration and schemas
│   └── actions/           # Server actions
├── public/                # Static assets
└── package.json           # Dependencies and scripts
```

### Step 1: Clone the Repository

<Snippet text="git clone https://github.com/vercel/ai-sdk-rag-starter" />

<Snippet text="cd ai-sdk-rag-starter" />

### Step 2: Install Dependencies

<Snippet text="npm install" />

<Note>
  The starter uses npm by default, but you can also use pnpm or yarn if you
  prefer.
</Note>

### Step 3: Environment Configuration

Copy the example environment file:

<Snippet text="cp .env.example .env" />

You'll need to configure these environment variables:

<CodeGroup>
  <CodeGroupItem title=".env">
    ```bash # Database connection (we'll set this up next)
    DATABASE_URL="postgresql://..." # OpenAI API key (we'll get this later)
    OPENAI_API_KEY="sk-..." ```
  </CodeGroupItem>
</CodeGroup>

## 2.2 PostgreSQL Database Setup with pgvector

**Duration**: 40 minutes

### Why PostgreSQL with pgvector?

<Columns cols={2}>
  <Card title="Vector Storage" icon="database">
    pgvector extension enables efficient vector storage and similarity search
  </Card>
  <Card title="Production Ready" icon="check">
    PostgreSQL is battle-tested and widely used in production environments
  </Card>
</Columns>
<Columns cols={2}>
  <Card title="Vercel Integration" icon="cloud">
    Seamless integration with Vercel's database offerings
  </Card>
  <Card title="Cost Effective" icon="dollar">
    More cost-effective than specialized vector databases for most use cases
  </Card>
</Columns>

### Option 1: Vercel Postgres (Recommended)

<Steps>
  <Step title="1. Create Vercel Account">
    Go to [vercel.com](https://vercel.com) and sign up or log in
  </Step>
  <Step title="2. Navigate to Storage">
    In your Vercel dashboard, go to the **Storage** tab
  </Step>
  <Step title="3. Create Database">
    Click **Create Database** and select **Postgres**
  </Step>
  <Step title="4. Configure Database">
    Choose your region and database name
  </Step>
  <Step title="5. Get Connection String">
    Copy the `DATABASE_URL` from the database settings
  </Step>
</Steps>

### Option 2: Local PostgreSQL Setup

If you prefer to run PostgreSQL locally:

<Snippet text="# macOS (using Homebrew)" />
<Snippet text="brew install postgresql" />
<Snippet text="brew services start postgresql" />

<Snippet text="# Create database" />
<Snippet text="createdb rag_tutorial" />

<Snippet text="# Install pgvector extension" />
<Snippet text="psql rag_tutorial -c 'CREATE EXTENSION IF NOT EXISTS vector;'" />

### Database Migration

Once your database is set up, run the initial migration:

<Snippet text="npm run db:migrate" />

This will:

- Add the pgvector extension to your database
- Create the initial `resources` table
- Set up the basic schema structure

### Verify Database Connection

Test your database connection:

<Snippet text="npm run db:studio" />

This opens Drizzle Studio, a web interface for managing your database. You should see:

- The `resources` table
- The pgvector extension enabled
- Ability to view and edit data

### Troubleshooting Common Issues

<Callout type="warning">
  **Connection Issues**: Make sure your `DATABASE_URL` is correct and the
  database is accessible
</Callout>

<Callout type="warning">
  **pgvector Extension**: Ensure the pgvector extension is installed and enabled
</Callout>

<Callout type="warning">
  **Migration Errors**: If migrations fail, check that your database user has
  the necessary permissions
</Callout>
