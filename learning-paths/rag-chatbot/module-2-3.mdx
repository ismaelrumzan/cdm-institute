## Starting the Development Server

<Steps>
  <Step title="1. Start the development server">
    Run the Next.js development server
  </Step>
  <Step title="2. Access the application">
    Open your browser and navigate to the local development URL
  </Step>
  <Step title="3. Verify functionality">
    Test the basic application features
  </Step>
</Steps>

<Snippet>
```bash
# Start the development server
npm run dev

# You should see output like:

# - ready started server on 0.0.0.0:3000, url: http://localhost:3000

# - event compiled client and server successfully

````
</Snippet>

<Callout type="info">
  **Development URL**: The application will be available at `http://localhost:3000`
</Callout>

<Callout type="info">
  **Hot Reload**: Any changes you make to the code will automatically refresh the browser
</Callout>

  </CodeGroupItem>
  <CodeGroupItem title="lib/db.ts">
```typescript
// Database configuration
// Sets up Drizzle ORM with PostgreSQL
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'

const client = postgres(process.env.DATABASE_URL!)
export const db = drizzle(client)

````
  </CodeGroupItem>
  <CodeGroupItem title="drizzle/schema.ts">
```typescript
// Database schema definition
// Defines tables and their relationships
import { pgTable, text, timestamp } from 'drizzle-orm/pg-core'

export const resources = pgTable('resources', {
  id: text('id').primaryKey(),
  content: text('content').notNull(),
  createdAt: timestamp('created_at').defaultNow(),
})
````

  </CodeGroupItem>
</CodeGroup>

## Troubleshooting Common Issues

<Callout type="warning">
  **Issue**: Development server won't start
  **Solution**: Check for port conflicts, ensure all dependencies are installed
</Callout>

<Callout type="warning">
  **Issue**: Database connection errors
  **Solution**: Verify DATABASE_URL in .env.local, ensure PostgreSQL is running
</Callout>

<Callout type="warning">
  **Issue**: TypeScript compilation errors
  **Solution**: Check for missing dependencies, verify type definitions
</Callout>

<Callout type="warning">
  **Issue**: Hot reload not working
  **Solution**: Check file permissions, restart the development server
</Callout>
